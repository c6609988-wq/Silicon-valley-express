## 三、每日采集时间

| 时间（北京） | UTC | 行为 |
|------|------|------|
| **07:00** | 23:00 前日 | 采集所有 X 源新推文，AI 评分选 Top3 入库，推送飞书 |
| **15:00** | 07:00 | 补充采集，若有新推文入库并推送，否则**静默跳过** |

### 07:00 推送逻辑（飞书）

```
有今日新推文 → 推今日 Top3
没有新推文   → 推昨日 Top3-6（博主还没发推）
昨日也没有   → 推数据库最近6条历史文章
```

### 15:00 推送逻辑（飞书）

```
有新推文 → 推当日新增 Top3
无新推文 → 静默跳过，不推送
```

## 四、AI 评分体系

每次采集后，将所有新推文**整体送给 AI 评分**，选出最有价值的 3 条。

### 评分维度（4个）

| 维度 | 说明 |
|------|------|
| **信息增量** | 是否包含新知识 / 新数据 / 产品发布 |
| **行业影响力** | 是否影响 AI / 科技行业走向 |
| **洞见深度** | 是否有独特观点或前瞻判断 |
| **实用性** | 是否对从业者有直接参考价值 |

### 过滤规则（采集前）

- 过滤转发（Retweet）
- 过滤纯回复（Reply）
- 过滤纯图片无文字推文
- 过滤正文少于 30 字的推文

### 博主预设评分（作为兜底权重）

分值 20 = 最高优先级，15 = 较低优先级。AI 实时评分为主，预设分数仅供参考排序。

## 五、AI 分析输出格式（短推文）

每条入库文章的 AI 分析包含三个部分：

### 一、今日核心要点
```
1. [话题关键词]：[一句话概括，不超过 30 字]
2. [话题关键词]：[一句话概括]
...（3-7 个要点，按重要性排序）
```

### 二、划重点
```
2-4 句话简短分析。
语气：像一个懂行的朋友说"今天这人说了啥"。
如果没有重要内容，直接说"今天主要是日常互动，没有重大信息"。
```

### 三、原文翻译
```
[时间] [「原创」/ 「转发」/ 「回复@xxx」]
原文：xxx
翻译：xxx
---
```

> App 内显示时，**只展示中文翻译部分**，其余两段显示在 AI 智能总结区域。

## 六、AI 分析输出格式（长内容 / YouTube）

适用于视频字幕、播客转录、RSS 长文。

### 一、标题
【信息来源】+【核心价值点】

### 二、引言
原文最有冲击力的一句话 + 核心内容概述

### 三、核心观点
逐层展开，二级论点 + 支撑逻辑 + 精彩引用

### 四、朋克张个人思考
由 AI 扮演资深 AI 产品经理"朋克张"，给出有态度的判断：
- 偶像：Peter Thiel、Naval、Sam Altman
- 要求：敢下判断，有逻辑支撑，不说套话

### 五、信息来源
原标题 / 作者 / 频道 / 发布时间

---

## 七、写作禁忌

AI 分析输出**明确禁止**出现以下内容：

- 套话：关键启示、值得注意的是、综上所述
- 成语滥用：立竿见影、醍醐灌顶
- 连接词滥用：首先/其次/最后
- 小红书体：宝子们、家人们、建议收藏
